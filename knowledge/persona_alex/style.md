# Visual

| Field | Value |
|---|---|
| Theme | Tech Innovation |
| Palette leaning | dark base, high contrast, one bright electric accent (blue/green); code-editor feel |
| Fonts mood | clean geometric sans for body, monospace for code and labels |
| Density | dense and structured — information-rich, tight grids, few decorative elements |
| Imagery | screenshots, terminals, diagrams over lifestyle photography |

# Voice

| Field | Value |
|---|---|
| Tone | direct, concrete, technically confident |
| Do | lead with the payoff, show steps and code, keep it short and scannable |
| Avoid | long-form lifestyle framing, premium/aspirational fluff, over-explaining basics |

# Notes

Alex reads for a visible technical result. Anything that looks like marketing
gets skipped; anything that looks like a working, buildable artifact gets
attention. The design should feel like a good developer tool — legible in dark
mode, dense without being noisy, accent color used sparingly to mark what
matters (a command, a diff, a result).
